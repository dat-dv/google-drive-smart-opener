# UI Specification

Ứng dụng chạy nền (background/menu bar app, xem R13). UI là cửa sổ "nhìn
vào" index sống, không phải nguồn dữ liệu — đóng UI không dừng watcher.

## Main window — Mapping Tree

Khu vực trung tâm là **cây thư mục đã ánh xạ** (Mapping Tree), hiển thị
song song 2 cột cho mỗi `FolderMapping`:

```
[Local]                          [Google Drive]
▾ ~/Projects/Finance        ↔    ▾ My Drive/Work/Finance        ✓ ACTIVE
    report.xlsx                      report.xlsx
    invoice-2024.pdf                 invoice-2024.pdf
▾ ~/Desktop/Scans            ↔    ▾ My Drive/Other/Scans         ⚠ DRIVE_DELETED
▸ ~/Projects/ClientX          ↔   (chưa map)                     — UNMAPPED
```

- Mỗi node folder cấp cao nhất = 1 `FolderMapping`. Mở rộng (expand) để
  xem các file/folder con (đọc từ index, không scan trực tiếp filesystem
  — R10).
- Badge trạng thái bên phải mỗi mapping: `ACTIVE` (xanh), `DRIVE_DELETED`
  / `LOCAL_MISSING` (vàng, cảnh báo), `CONFLICT` (đỏ, nếu có document con
  đang conflict).
- Click vào 1 file trong cây → mở theo Open Workflow hiện có (ưu tiên bản
  Drive).

## Thao tác CRUD trên Mapping Tree (R14)

- **+ Add Mapping** (nút trên toolbar hoặc click phải vào vùng trống):
  mở dialog chọn 1 folder local + 1 folder Drive → tạo `FolderMapping`
  mới. App index lại toàn bộ subtree ngay sau khi tạo.
- **Edit Mapping** (click phải vào 1 mapping → "Edit"): cho phép đổi
  `localFolderPath` hoặc `driveFolderPath`. App re-index subtree liên
  quan, giữ nguyên `id` của mapping và `id` của các `Document` con khi có
  thể (match theo checksum/relative path) để không làm mất lịch sử.
- **Delete Mapping** (click phải → "Remove Mapping"): dialog xác nhận với
  2 lựa chọn:
  - "Chỉ gỡ ánh xạ" (mặc định) — `FolderMapping` bị xoá, các `Document`
    con chuyển `folderMappingId = NULL`, status không đổi, file vật lý
    giữ nguyên cả 2 phía.
  - "Gỡ ánh xạ và xoá index document con" — xoá luôn các `Document`
    record con khỏi DB (vẫn KHÔNG đụng tới file vật lý).
- **Cảnh báo cấu trúc thay đổi** (theo R15): khi 1 mapping chuyển sang
  `DRIVE_DELETED` hoặc `LOCAL_MISSING`, hiển thị banner ngay trên dòng
  mapping đó kèm action:
  - `DRIVE_DELETED`: "Drive folder đã bị xoá hoặc đổi tên" → nút
    [Xem trên Drive] / [Khôi phục nếu còn trong Trash] / [Gỡ mapping].
  - Drive folder bị rename/move (không xoá): "Tên/đường dẫn Drive đã
    thay đổi, local chưa cập nhật theo" → nút [Đồng bộ tên local theo
    Drive] (opt-in, xem R15) / [Bỏ qua].
  - `LOCAL_MISSING`: "Không tìm thấy folder local gốc" → nút [Chọn lại
    folder local] / [Gỡ mapping].

## Các khu vực khác

- **Recent Documents** — danh sách file vừa mở gần đây (phẳng, không
  theo cây), kèm trạng thái LINKED / CONFLICT...
- **Status** — tổng quan: số mapping ACTIVE, số document CONFLICT, số
  cảnh báo đang chờ xử lý.
- **Index Statistics** — số lượng document đã index, số folder mapping,
  lần rebuild gần nhất.
- **Google Drive Status** — kết nối / đường dẫn / dung lượng.
- **Conflicts** — danh sách document đang ở trạng thái CONFLICT, có nút
  xử lý nhanh (mở dialog R9).
- **Settings** — xem mục Settings.

## Dialogs (giữ nguyên từ workflow mở file)

1. **Similar document found** (R3.1)
   - "Open Google Drive Version" / "Import As New Copy"
2. **Multiple candidates** (R3.2)
   - Picker list + "Create New Copy"
3. **Conflict detected** (R9)
   - 6 lựa chọn: Open Drive / Open Local / Replace Drive / Replace Local /
     Create New Copy / Cancel

## Settings panel

- Google Drive folder path
- Import folder (`My Drive/Other`, có thể đổi)
- Conflict strategy (always-ask / prefer-drive / prefer-local)
- Auto import (bật/tắt)
- Auto-sync tên local theo Drive khi Drive rename/move folder
  (mặc định: TẮT — xem R15, hành động này luôn opt-in)
- Theme (light/dark/system)
- Rebuild index (button, có confirm dialog vì tốn thời gian)
